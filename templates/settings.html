hosted features



Adds an unsubscribe link and header to outgoing emails, and a hosted "most recent newsletter" page. Requires the instance to be reachable at the URL below by your recipients. Once someone unsubscribes it always stays honored, even if this is later disabled.




The externally-reachable URL recipients use for unsubscribe links and the hosted newsletter page (and for hosted images, if enabled below). Required when Hosted Features are enabled.




Useful if your hosted images need a publicly-reachable URL (e.g. for mail providers that proxy-fetch images) but you'd rather keep the unsubscribe and "view latest newsletter" links on a private URL, such as one only reachable over your own tailnet/VPN. Requires Hosted Features to be enabled.




The URL used for unsubscribe links and the "view latest newsletter" link instead of Hosted Base URL. Required when this is enabled.




Serves poster art via tokenized public URLs instead of embedding it in each email, which shrinks message size. This exposes a proxy of your Plex art publicly (behind non-guessable per-image tokens) only enable if you're comfortable with that tradeoff. Requires Hosted Features to be enabled.




How long hosted images are kept on disk before being deleted. Applies only when Hosted Images is enabled.
